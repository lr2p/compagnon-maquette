Analyse technique : livrer XMO en application native

Faisabilité, difficulté et contraintes de chaque fonctionnalité, avec pour objectif une distribution Mac App Store, .dmg (Mac) et .exe (Windows).

Verdict en une phrase : tout est faisable, mais l'app native ne remplace pas le cloud — elle l'habille. Le canal principal recommandé est le .dmg notarié + .exe signé (liberté totale), avec le Mac App Store en second canal de confiance dans une édition conforme (sans « Mes avis » rémunérés, paiement sur le web). Le chemin critique n'est pas le code : ce sont les validations Google (Gmail), à lancer dès la semaine 1.

1 · Le point d'architecture qui conditionne tout

Une app installée ne peut pas contenir l'intelligence : elle contient le cockpit.

🖥️ L'app XMO (le cockpit)

  • Interface univers + pastille + télé incrustée
  • Plein écran « kiosque », démarrage automatique
  • Caméra, micro, lecture à voix haute (voix système)
  • Surveillance du dossier Brouillons local
  • Notifications, mises à jour automatiques
⇄

☁️ Le cloud XMO (le cerveau)

  • Synchronisation des boîtes mail (jetons chiffrés)
  • Bouclier anti-arnaque + résumés + rangement IA (LLM via OpenRouter)
  • Console aidant (web) + appairage + alertes
  • Push de contenu (médias choisis par la famille)
  • Facturation Stripe
Pourquoi pas « tout dans l'app » ? ① Une clé API (OpenRouter) embarquée dans un binaire est extractible en quelques minutes → elle doit rester côté serveur. ② Les modèles locaux tournant sur le Mac d'un senior sont trop faibles pour un anti-arnaque fiable. ③ La console aidant est de toute façon un site web. ④ Améliorer les prompts et le routing des modèles sans re-publier l'app (donc sans re-review App Store) exige un backend. Modèle Netflix/Spotify : l'app est un client, le service est en ligne.

2 · Les trois canaux de distribution

Le même code (coquille Tauri 2 recommandée : ~10 Mo, webview système, builds Mac + Windows ; alternative Electron, plus lourde mais plus mature) produit les trois livrables.

Canal principal Mac

.dmg notarié

Coût : Apple Developer 99 $/an (Developer ID + notarisation) Délai : notarisation en minutes, aucune review Contraintes : Gatekeeper (glisser dans Applications — l'aidant installe), auto-update à intégrer (Sparkle) Liberté : totale — tous les univers, paiement Stripe, 0 % de commission
✅ À faire en premier. Aucune règle d'Apple sur le contenu.
Canal principal Windows

.exe signé (+ Microsoft Store)

Coût : certificat de signature OV/EV ~300–500 €/an Délai : immédiat ; SmartScreen fait la tête au début avec un OV (réputation à construire), pas avec un EV Contraintes : installeur (MSI/NSIS) + updater (WinSparkle) ; Microsoft Store en complément = confiance + MàJ auto, review légère
✅ Même liberté que le .dmg. Le Store Microsoft est un bonus confiance facile.
Canal 2 — confiance

Mac App Store

Coût : inclus dans les 99 $/an + 15–30 % de commission si paiement intégré Délai : review 1–7 j à chaque mise à jour Contraintes dures : sandbox obligatoire (OK pour XMO) ; « Mes avis » rémunéré interdit (rémunérer le visionnage de pubs viole les guidelines) ; les wrappers web (réseaux sociaux en iframe) mal vus (règle 4.2)
🟡 À faire ensuite, en édition conforme : login seul (abonnement acheté sur le web par l'aidant → pas de commission), sans l'univers Avis.

4 · Les quatre murs à connaître (aucun n'est du code)

① Google / Gmail — le chemin critique

Lire les mails Gmail d'un utilisateur = « scopes restreints » : vérification d'app + audit de sécurité CASA annuel. Délai en semaines/mois, coût 0–5 k€ selon le laboratoire.

Parade : déposer le dossier Google dès la semaine 1 et développer avec Outlook/IMAP en attendant.

② Meta — publier sur un profil perso est impossible

Aucune API ne poste sur le Facebook/Instagram personnel d'un utilisateur. Ce n'est pas une difficulté, c'est une porte fermée.

Parade : WhatsApp en 1 clic (ça, ça marche), YouTube API, compte Creator configuré par l'aidant, ou relais famille.

③ Droits TV & musique

Diffuser TF1 ou Deezer dans la télé incrustée exige des accords (et l'API Deezer est fermée). Le calendrier appartient aux ayants droit.

Parade : V1 = radios + presse + YouTube + chaînes à flux ouverts (Arte, France 24…). Accords en parallèle.

④ App Store vs « Mes avis »

Rémunérer le visionnage de pubs est contraire aux guidelines Apple : l'univers Avis (version cash) est incompatible avec le canal App Store.

Parade : deux éditions du même code — complète (.dmg/.exe) et conforme (App Store, sans Avis) ; ou cagnotte 100 % don.

Contrainte transversale — RGPD : les mails d'un senior contiennent potentiellement des données de santé → hébergement UE, chiffrement au repos, registre des traitements, droit à l'oubli, et un mot dans le contrat aidant. À intégrer dès l'architecture, pas après.

5 · Chiffrage indicatif (V1 native, ~4 mois)

Équipe : 1 dev fullstack senior + toi. Les blocs se parallélisent ; le calendrier est dicté par Google, pas par le code.

  • Sem. 1⚠️ Dépôt des dossiers Google (Gmail) et Microsoft — chemin critique, tout de suite
  • Sem. 1–4Coquille Tauri : cockpit, univers, pastille, télé PiP, kiosque, updater, signature/notarisation .dmg + .exe
  • Sem. 3–8Connecteurs mail (Outlook + IMAP d'abord, Gmail à la validation) + sync serveur + chiffrement des jetons
  • Sem. 5–8Bouclier anti-arnaque + résumés + lecture voix (routing petit modèle via OpenRouter)
  • Sem. 7–9Rangement IA des brouillons + dossiers + photos
  • Sem. 8–12Console aidant web : appairage, alertes bouclier, agenda interne, contacts famille, Stripe
  • Sem. 12–14Médias light + famille (liens d'appel) + QA avec de vrais binômes senior/aidant
  • Sem. 14–16Durcissement, RGPD, bêta fermée ; soumission Mac App Store (édition conforme) ensuite

6 · Recommandations

  1. Coquille Tauri 2 (léger, un code → .dmg + .exe + App Store) ; le front cockpit existe déjà en HTML.
  2. Ordre des canaux : ① .dmg notarié + .exe signé (liberté, 0 % commission) → ② Microsoft Store (confiance facile) → ③ Mac App Store en édition conforme.
  3. Paiement toujours sur le web (Stripe, par l'aidant) — jamais d'achat intégré Apple.
  4. Lancer les validations Google semaine 1 — c'est elles qui datent la sortie, pas le développement.
  5. Transparence sur Meta et les Avis : la publication sur un profil FB/IG personnel est impossible par API (alternatives : WhatsApp, YouTube, relais famille) ; l'univers Avis en version rémunérée est incompatible avec l'App Store (piste élégante : cagnotte 100 % don).
